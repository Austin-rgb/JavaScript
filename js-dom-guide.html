DOM

What is the DOM?

When a browser loads an HTML page, it turns the markup into a tree of objects in memory. This tree is called the Document Object Model — the DOM. JavaScript can read this tree, change it, add to it, and delete from it. Every change shows up immediately on the page.

document
└─ <html>
├─ <head>
│ └─ <title> "My Page" </title>
└─ <body>
├─ <h1 id="title"> "Hello" </h1>
└─ <p class="intro"> "Welcome" </p>

Every box in that tree is a node. Element nodes (<h1>, <p>) are the most common ones you'll work with. The entry point to the whole tree is document — a global object that's always available in any script running in the browser.

🧠

document vs window: window is the browser tab itself — it holds globals like alert(), setTimeout(), and location. document lives inside window and represents the page content. Most DOM work uses document.

Selecting

Selecting elements

Before you can change anything, you need to grab a reference to it. These are the four methods you'll use:

MethodFindsReturns
getElementById('id') One element with that id Single element or null
querySelector('css') First element matching CSS selector Single element or null
querySelectorAll('css') All elements matching CSS selector NodeList (array-like)
getElementsByClassName('cls') All elements with that class HTMLCollection (live) 
// getElementById — fastest, use when you have an id
const title = document.getElementById('title');

// querySelector — most flexible, takes any CSS selector
const firstBtn  = document.querySelector('button');
const heroHeading = document.querySelector('#hero h2');
const active     = document.querySelector('.active');

// querySelectorAll — get many elements at once
const allButtons = document.querySelectorAll('button');
const allCards   = document.querySelectorAll('.card');

// Loop over a NodeList just like an array
allButtons.forEach(btn => {
  console.log(btn.textContent);
});
⚠

Always check for null. If the selector finds nothing, querySelector returns null. Calling a method on null throws a TypeError. Either confirm the element exists in HTML, or check: if (el) { ... }.

Manipulating

Manipulating elements

Once you have a reference to an element, you can read or change almost anything about it: its text, its HTML content, its attributes, or even whether it exists on the page at all.

Reading and writing content:

const el = document.getElementById('box');

// textContent — plain text only, safe (no HTML injection)
el.textContent = 'Hello!';
console.log(el.textContent);

// innerHTML — renders HTML tags (use carefully — never with user input)
el.innerHTML = '<strong>Bold text</strong> inside the element';

Creating and inserting new elements:

// 1. Create a new element
const newItem = document.createElement('li');
newItem.textContent = 'New item';
newItem.classList.add('item');

// 2. Insert it into the page
const list = document.getElementById('my-list');
list.appendChild(newItem);       // adds to end
list.prepend(newItem);           // adds to start

// insertAdjacentElement gives precise control
// positions: 'beforebegin' 'afterbegin' 'beforeend' 'afterend'
list.insertAdjacentElement('beforeend', newItem);

Removing elements and working with attributes:

// Remove an element from the page
const staleEl = document.getElementById('remove-me');
staleEl.remove();

// Get / set / remove HTML attributes
const link = document.querySelector('a');
console.log(link.getAttribute('href'));
link.setAttribute('href', 'https://example.com');
link.setAttribute('target', '_blank');
link.removeAttribute('target');

Events

Events — responding to the user

Events are things that happen in the browser: a click, a key press, the page loading, a mouse moving. You attach a listener to an element, and it calls a function whenever that event fires.

const btn = document.getElementById('my-btn');

// addEventListener(eventName, callbackFunction)
btn.addEventListener('click', function(event) {
  console.log('Button clicked!');
  console.log('Event object:', event.type);
});

// Arrow function — same result, cleaner
btn.addEventListener('click', (e) => {
  btn.textContent = 'Clicked!';
  btn.disabled = true;
});

The most common events:

EventFires when…
clickUser clicks the element
dblclickUser double-clicks
mouseover / mouseoutMouse enters / leaves the element
keydown / keyupA key is pressed / released
inputValue of an input changes
submitA form is submitted
DOMContentLoadedHTML has fully parsed (before images load)

Event delegation — instead of attaching a listener to every item, attach one to the parent and check which child was clicked. This is essential when items are added dynamically.

const list = document.getElementById('task-list');

// One listener on the parent handles all children
list.addEventListener('click', (e) => {
  // e.target is the specific element that was clicked
  if (e.target.tagName === 'LI') {
    e.target.classList.toggle('done');
  }
});

Forms

Forms & input

Forms are how users send data to your page. The key workflow is: listen for submit, prevent the default page reload with e.preventDefault(), then read the values from the inputs.

const form  = document.querySelector('form');
const input = document.getElementById('username');
const output = document.getElementById('result');

form.addEventListener('submit', (e) => {
  e.preventDefault(); // stops the browser refreshing the page

  const name = input.value.trim(); // .trim() removes whitespace

  if (!name) {
    output.textContent = 'Please enter a name.';
    return;
  }

  output.textContent = `Hello, ${name}!`;
  input.value = ''; // clear the input
});

Reading different input types:

// Text input — .value gives the current string
const text = document.getElementById('name-input').value;

// Number input — .value is still a string, convert it
const age = Number(document.getElementById('age-input').value);

// Checkbox — .checked is a boolean
const agreed = document.getElementById('agree').checked;

// Select / dropdown — .value gives selected option
const colour = document.getElementById('colour-select').value;

Styling

Dynamic styling

There are two ways to change how something looks with JavaScript. Prefer classList — it keeps your style logic in CSS where it belongs, and JavaScript just toggles class names.

const el = document.getElementById('card');

// classList — the right way
el.classList.add('active');       // add a class
el.classList.remove('hidden');    // remove a class
el.classList.toggle('open');      // add if absent, remove if present
el.classList.contains('active'); // → true or false

// .style — fine for one-off, calculated values
el.style.width  = '200px';
el.style.opacity = '0.5';
// camelCase for properties with hyphens:
el.style.backgroundColor = '#3b82f6';  // background-color
el.style.borderRadius = '8px';         // border-radius
💡

classList vs style: put all your visual states in CSS classes (.active, .hidden, .error), then use classList.toggle() to switch between them. Only use .style directly for values you calculate at runtime (like a width based on a number).

localStorage

localStorage — saving data in the browser

localStorage lets you save small pieces of data in the user's browser that survive page reloads and even browser restarts. It stores everything as strings, so use JSON.stringify/JSON.parse for anything more than a simple value.

// Save a value — stays until explicitly cleared
localStorage.setItem('username', 'Ada');

// Read it back
const name = localStorage.getItem('username');
console.log(name); // "Ada"

// Remove one key
localStorage.removeItem('username');

// Clear everything
localStorage.clear();

// Saving objects — must stringify and parse
const settings = { theme: 'dark', fontSize: 16 };
localStorage.setItem('settings', JSON.stringify(settings));

const loaded = JSON.parse(localStorage.getItem('settings'));
console.log(loaded.theme); // "dark"
⚠

localStorage limits: stores only strings, has a ~5MB limit per origin, and is synchronous (can block the page if overused). Never store passwords, tokens, or sensitive data here — it's accessible to any JS on the page.
